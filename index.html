Cube

An easy way to manage your locally installed Caché and Ensemble Instances on OS X

Download Cube

Features

  • Start/Stop/Restart Instances
  • Auto-start an Instance on system start-up
  • Telnet to an Instance
  • Open up Management Portal in your web browser
  • Open up the directory the Instance is installed to using Finder
  • Open up Documentation and Class Reference in your web browser
  • Auto-start Cube when you login to your OS X account

Installing

  1. Uninstall any previous installations of Cube
  2. Download the ZIP file and unzip contents
  3. Telnet to an Instance
  4. Move Cube.app into your Applications directory
  5. Double-click Cube.app to start it
  6. You will be prompted for your password because Cube needs to install a privileged helper
    IMAGE
  7. Cube should now appear in your status menu
    IMAGE

Uninstalling

  1. Make sure "Start Cube at Login" is disabled
  2. Quit Cube app
  3. Delete the following files in Terminal (will require sudo): /Library/PrivilegedHelperTools/com.InterSystems.CubeHelper /Library/LaunchDaemons/com.InterSystems.CubeHelper.plist
  4. Restart your system

FAQ

Why am I being prompted for my password?

You are allowing Cube to install the privileged helper to your system's library directory. You should only be prompted for your password once.

What is a privileged helper?

A utility deployed with Cube that has root privileges required for:

  • Stopping/Starting an Instance
  • Installing system start-up scripts for Instances
What version of Cube do I have?

A utility deployed with Cube that has root privileges required for:

  • Right-click on Cube.app from Applications directory
  • Select Get Info
  • Record the Version number
